# Jaro / Jaro–Winkler на C++ (sequential + OpenMP)

Проект с реализациями метрики **Jaro** и **Jaro–Winkler** и утилитами для:
- вычисления схожести **по парам строк из датасета** (CSV/`|`);
- **бенчмарка** времени выполнения (последовательно / OpenMP);
- поиска похожего **слова в тексте** по порогу схожести (есть sequential и аналогичная параллельная версия, если файл присутствует).

## Что реализовано

### A) Jaro / Jaro–Winkler (база)
Реализованы:
- `jaro_similarity(s1, s2)`
- `jaro_winkler_similarity(s1, s2)`

Параметры через структуру `JWOptions`:
- `case_insensitive` — сравнение без учета регистра (ASCII)
- `p` — коэффициент Винклера (обычно 0.1)
- `max_prefix` — максимальная длина префикса, участвующая в «boost» (обычно 4 или 8)

### B) Бенчмарк по парам строк
Есть две версии:
- **sequential**: считает сумму `jaro_winkler_similarity()` по N парам строк и усредняет время по `num_runs`.
- **OpenMP**: сравнивает sequential vs parallel, печатает:
  - среднее время,
  - speedup,
  - efficiency,
  - поддерживает планирование `guided|dynamic|static` + `chunk`,
  - диапазон потоков `tmin..tmax`.

### C) Поиск похожего слова в тексте
- Загружает слова из текстового файла (разделение по пробелам), обрезает пунктуацию по краям токенов.
- Ищет **первое** слово, у которого `JW(query, word) >= threshold`.
- Замеряет время выполнения.

## Кратко про алгоритм

### Jaro
1) Определяет **окно поиска совпадений** для каждого символа:  
   `range = floor(max(|s1|, |s2|)/2) - 1` (не меньше 0)
2) Считает количество совпадений `m` в пределах окна.
3) Строит строки совпавших символов `m1`, `m2`.
4) Считает транспозиции `t` (половина количества несовпадений по позициям в `m1/m2`).
5) Итог:
\[
J = \frac{1}{3}\left(\frac{m}{|s1|}+\frac{m}{|s2|}+\frac{m-t}{m}\right)
\]

### Jaro–Winkler
Добавляет «бонус» за общий префикс длины `l`:
\[
JW = J + l \cdot p \cdot (1 - J)
\]
где `l <= max_prefix`.

## Структура проекта

Реальная структура может отличаться (смотри дерево в репозитории), но логика обычно такая:

- `jw.cpp` — **sequential**: Jaro–Winkler по парам + среднее время
- `j-w_omp.cpp` (или похожее имя) — **OpenMP** бенчмарк по парам, speedup/efficiency, schedule/chunk
- `jw_word_search.cpp` — поиск похожего слова в тексте (sequential)
- `jw_word_search_omp.cpp` — аналогичный поиск, но параллельный (если присутствует)
- `data/` — датасеты (CSV/текст)

## Требования

- Компилятор C++ с поддержкой **C++17**:
  - `g++` (GCC) или `clang++` (Clang)
- Для OpenMP:
  - GCC: `-fopenmp`
  - Clang: иногда нужен `libomp` (в Linux: `sudo apt install libomp-dev`), и флаги могут отличаться
- ОС: Linux/WSL/Linux native/Windows (MinGW) — главное чтобы OpenMP работал.
